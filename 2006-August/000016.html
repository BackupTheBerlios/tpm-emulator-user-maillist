On Friday, 11. August 2006 13:45, Carolin Latze wrote:
> I recognized a strange behavior of the latest stable release (0.4). When
> I load the module using the "clear" startup parameter, the tpm is not
> resetted as expected. [...]
> Does this mean, that is not desired anymore to clean the whole tpm when
> using the "clear" option? (now, the only way to reset the tpm completely
> is to remove the files in /var/tpm)
To completely reset the TPM emulator you either have to manually remove
the files located in /var/tpm or deactivate and reload it:
# modprobe tpm_emulator startup="deactivated"
# rmmod tpm_emulator
# modprobe tpm_emulator startup="clear"
