Using mknod shouldn't be necessary on current udev-based
systems. In order that the device is created automatically
whenever the module is loaded, all you have to do is define
an according rule file in /etc/udev/rules.d/
(see http://www.reactivated.net/writing_udev_rules.html).
In summary: create a file called

/etc/udev/rules.d/70-tpmd_dev.rules

with the content

KERNEL=="tpm", NAME="%k", SYMLINK+="tpm0", GROUP=tss, MODE="0660"

(a template is available in the directory tpmd_dev/).
