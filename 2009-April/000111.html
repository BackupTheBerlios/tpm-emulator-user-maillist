>    I want to use the fuction Tddli_TransmitData for a serial random data.In test_tddl.c,
>    there is a example of using Tddli_TransmitData to send a reset  command.I want
>    to know how to fill the first parameter pTransmitBuf.Could you help me to slove the problem?
The transmit buffer contains a marshaled TPM command as specified
by the TPM specification [1]. Note, however, that the TPM emulator
internally uses the random source of the Linux kernel by default;
this random source can be accessed much easier via the /dev/random
or /dev/urandom device file.

Regards,
Mario

[1] https://www.trustedcomputinggroup.org/specs/TPM/mainP3Commandsrev103.pdf
