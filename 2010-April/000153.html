I cannot reproduce the error on my system. Does it remain
with trousers version 0.3.4? Please note that trousers stores
some system data in /var/lib/tpm/ which might become
invalid if you use the TPM (emulator) with other tools
that bypass the TSS API (i.e. trousers) or if you change
the internal state of the TPM by other means. Thus, stopping
the daemons and removing all files stored in /var/lib/tpm/
might help.
