On Wednesday, 01. October 2008 18:49:01 John Lyle wrote:
> I'm having some difficulty using the TPM Emulator for certifying keys 
> bound to PCR values.  
> [...]
>  * The TPM_PCR_INFO structure returned appears to be given the wrong 
> length (PCRInfoSize in the TPM_CERTIFY_INFO structure).  It reports at 
> 56 bytes but ought (I think) to be 44.
Since version 1.2 of the TPM specification, the data returned
by TPM_CertifyKey depends on the key to be certified. If the
PCRInfo structure of the respective key contains at most
16 PCR values, the old TPM_TAG_CERTIFY_INFO is used;
otherwise, if the key usage depends on more than 16 PCR
values, the new TPM_TAG_CERTIFY_INFO2 structure is used.
Could you please verify that the result is a valid
TPM_TAG_CERTIFY_INFO2 structure (or send me a hex-dump
of the result so that I can do it)? Thanks a lot.

Mario
